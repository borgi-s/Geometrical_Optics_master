**Date**: 2026-06-13
**Status**: draft for review

## 1. Problem

The dislocation/displacement stack is FCC/cubic-hardcoded in ways that 4.1/4.2
did not touch (they generalized the *cell*, not the *dislocations*):

- **Slip tables**: `_SLIP_SYSTEM_111` (12 FCC systems, `forward_model.py:107`),
  `_BASIS_TABLE` + `burgers_vectors()` with a literal `/ŌłÜ2` (FCC Ō¤©110Ō¤® length,
  `burgers.py:21,57`), `_ALL_111_PLANES` (4 planes, `orchestrator.py:1006`).
- **Burgers magnitude**: scalar `BURGERS_VECTOR = 2.862e-4 ┬Ąm` (Al,
  `constants.py:21`), an isotropic prefactor in every displacement function;
  the `*ŌłÜ2` reconstitution of integer indices is an FCC assumption.
- **Elasticity**: scalar `POISSON_RATIO = 0.334` (Al, `constants.py:24`),
  isotropic-elastic only (no `C_ijkl` ŌĆö explicitly out of scope, v3.x).
- **q_hkl**: 4 sites compute `q/|q|` ("B_0=I", `forward_model.py:268,689,1278,
  1355`) ŌĆö correct for cubic (incl. BCC), wrong for non-cubic (HCP).

The general cell geometry already exists and is the bridge: `UnitCell.A`
(real-space cell, columns a,b,c), `UnitCell.B` (= 2ŽĆ┬Ęinv(A)ßĄĆ, reciprocal),
`UnitCell.d_spacing` (metric `2ŽĆ/|B┬ĘG|`), and the non-cubic lab-frame q
construction in `oblique.compute_omega_eta` (`C_s_inv_T @ G_hkl`).

## 2. Approach overview

A data-driven **slip-system registry** keyed by *structure family*
(`fcc` / `bcc` / `hcp` / user-defined), derived from the space group or set
explicitly, replacing the three hardcoded tables. Burgers magnitude derived
from the cell. ╬Į from a config knob + table. Cubic paths (FCC **and** BCC)
stay in the orthonormal frame and the existing displacement/numba kernels are
untouched. HCP (4.3b) adds the non-orthonormal frame conversion and
per-dislocation |b|.

**Bit-identity gate (both stages):** the FCC default path ŌĆö no `structure_type`,
no `space_group`, no `cif` ŌĆö must remain byte-identical to current main. This
is the primary regression guard (existing FCC tests are the anchors).

## 3. Stage 4.3a ŌĆö BCC (cubic, this plan)

### 3.1 New module `crystal/slip_systems.py`

The single source of truth for slip geometry. Pure, no I/O, no gemmi.

```
StructureType = Literal["fcc", "bcc", "hcp"]   # extensible

@dataclass(frozen=True)
class SlipFamily:
    name: str                      # e.g. "{110}<111>"
    plane_family: tuple[int,int,int]   # representative (hkl), e.g. (1,1,0)
    burgers_family: tuple[int,int,int] # representative <uvw>, e.g. (1,1,1)

# Registry: structure -> ordered list of SlipFamily
_REGISTRY: dict[str, tuple[SlipFamily, ...]] = {
    "fcc": (SlipFamily("{111}<110>", (1,1,1), (1,1,0)),),
    "bcc": (SlipFamily("{110}<111>", (1,1,0), (1,1,1)),
            SlipFamily("{112}<111>", (1,1,2), (1,1,1)),),   # 4.3a scope decision below
}
```

Public functions (cubic-only in 4.3a; signatures accept the cell for 4.3b):

- `slip_systems(structure: str, *, families: Sequence[str] | None = None)
  -> list[SlipSystem]` where `SlipSystem` carries `(b, n, t)` **integer
  Miller** plus the symbolic family name. Generated by enumerating all
  sign/permutation variants of `plane_family` and `burgers_family` under the
  cubic point group, filtering to `b┬Ęn == 0` (glide), and deduplicating
  ┬▒(b,n) pairs by a canonical sign rule. `t` is `n ├Ś b` (cubic Cartesian).
- `plane_normals(structure, *, families=None) -> list[tuple[int,int,int]]` ŌĆö
  the distinct slip-plane normals (replaces `_ALL_111_PLANES`).
- `burgers_in_plane(structure, plane, *, families=None) -> np.ndarray` ŌĆö
  shape (m, 3) **unit** Burgers directions in that plane, with their negatives
  (replaces `_BASIS_TABLE` + `burgers_vectors`). Normalization is per-vector
  Euclidean (`v/|v|`), NOT the literal `/ŌłÜ2` ŌĆö bit-identical for FCC Ō¤©110Ō¤®,
  correct for BCC Ō¤©111Ō¤®.
- `derive_structure_type(*, structure_type, space_group, lattice) -> str`:
  **precedence** ŌĆö when a space group is present it is authoritative (the
  crystallographic evidence overrides a possibly-wrong user assertion): derive
  from centering letter + crystal system (F-cubicŌåÆfcc, I-cubicŌåÆbcc,
  P-hexagonalŌåÆhcp; via gemmi's `SpaceGroup` centring +
  `space_group_crystal_system`, called through a thin helper in `crystal/cif.py`
  so gemmi stays behind that boundary) and **raise if an explicit
  `structure_type` contradicts it**. With no space group, explicit
  `structure_type` wins over the default; with neither, `"fcc"` (back-compat).
  Unknown/unsupported ŌåÆ `ValueError` naming the choices.

**Enumeration validation (test-locked counts):** fcc {111}Ō¤©110Ō¤® ŌåÆ 12 systems,
4 planes; bcc {110}Ō¤©111Ō¤® ŌåÆ 12 systems, 6 planes; bcc {112}Ō¤©111Ō¤® ŌåÆ 12 systems,
12 planes; **bcc default ({110}+{112}) ŌåÆ 24 systems, 18 planes**. The
enumerator output for fcc {111}Ō¤©110Ō¤® must equal the current
`_SLIP_SYSTEM_111` as a **set** of (b╠é, n╠é, t╠é) up to the sign convention
(asserted in tests; this is how we prove the registry reproduces the hand
table before deleting it).

**4.3a families decision (Sina 2026-06-13):** ship bcc `{110}<111>` **and**
`{112}<111>` together as the default ŌĆö **24 systems total** (12 + 12) over
18 distinct planes (6 {110} + 12 {112}). `{123}<111>` deferred (rarely
needed; the user-defined `[[crystal.slip_system]]` escape hatch covers it).
`slip_families` can still narrow the default to a single family for studies
that want only one mode.

### 3.2 Burgers magnitude from the cell

- `burgers_magnitude(structure, family, cell: UnitCell) -> float`:
  the full lattice translation length of the slip direction, in the same
  ┬Ąm units as `BURGERS_VECTOR`. FCC ┬ĮŌ¤©110Ō¤® = a/ŌłÜ2; BCC ┬ĮŌ¤©111Ō¤® = aŌłÜ3/2;
  computed as `|A ┬Ę b_int| ┬Ę scale` where `b_int` is the integer Burgers
  vector and `scale` is the ┬Į for the centered lattice translation
  (`A` from `UnitCell`, converted mŌåÆ┬Ąm). For cubic this reduces to the
  closed forms above; the Al FCC value reproduces `2.862e-4 ┬Ąm` exactly
  (regression check).
- `BURGERS_VECTOR` in `constants.py` is retained as the FCC-Al default
  (back-compat: the default path passes it explicitly), but the population
  builders now source |b| from `burgers_magnitude` when a structure/cell is
  resolved. In 4.3a |b| is still a **scalar per population** (BCC ┬ĮŌ¤©111Ō¤® is
  single-valued); per-dislocation |b| arrives in 4.3b for HCP.

### 3.3 Poisson ratio ╬Į

- `crystal/elasticity.py` (new): `_POISSON_TABLE: dict[str, float]` keyed by
  element symbol (Al 0.334, Fe 0.29, W 0.28, Cu 0.34, Ni 0.31, Ti 0.32,
  Mg 0.29, ŌĆ”). **Each value carries an inline source citation** in the module
  docstring (Sina 2026-06-13) ŌĆö a polycrystalline/handbook reference per
  entry (e.g. *Kaye & Laby Tables of Physical & Chemical Constants*;
  *Simmons & Wang, Single Crystal Elastic Constants, 1971* for the
  single-crystal-averaged values). No bare numbers. Provenance also records
  which source/material produced the ╬Į used in a run.
  `poisson_ratio(*, override, material) -> float`: explicit `override` wins;
  else table lookup by `material`; else 0.334 (Al) ŌĆö the value used is emitted
  in provenance.
- `[crystal] poisson_ratio` (float) and `[crystal] material` (str, optional)
  added to the mount TOML surface; `POISSON_RATIO` constant retained as the
  fallback literal.

### 3.4 Config surface

Add to the shared mount-key set `_CRYSTAL_MOUNT_KEYS` (`config.py:72`) and
parse in `_crystal_mount_from_toml` (`kernel.py:58`):

```toml
[crystal]
structure_type = "bcc"        # optional; else derived from space_group/cif; else fcc
material = "Fe"               # optional; ╬Į table lookup
poisson_ratio = 0.29          # optional; overrides material/default
slip_families = ["{110}<111>"]  # optional; default per structure
```

- `CrystalMount` gains `structure_type: str | None`, `material: str | None`,
  `poisson_ratio: float | None`, `slip_families: tuple[str, ...] | None`
  (all default None ŌåÆ fully back-compat; `__post_init__` validates
  `structure_type` against the registry and against the derived value when a
  space group is also present, erroring on contradiction e.g.
  `structure_type="bcc"` + an F-cubic space group).
- **User-defined slip systems escape hatch**: `[[crystal.slip_system]]` array
  of tables, each `plane = [h,k,l]`, `burgers = [u,v,w]`, fed straight into the
  registry as an ad-hoc structure for exotic cases. Validated `b┬Ęn==0`.
- Identify path: `IdentificationCrystalConfig` validation (`config.py:822`,
  the `{111}`-only gate) switches to "the plane is a registered slip plane for
  the resolved structure"; `sweep_all_slip_planes` consults
  `plane_normals(structure)`.

### 3.5 Consumer rewrites (cubic-safe, FCC bit-identical)

- `forward_model.py` `_SLIP_SYSTEM_111` ŌåÆ `slip_systems(structure)` resolved
  once per run from the mount; `random_dislocations` draws over the resolved
  list; `wall` mode's hardcoded system ŌåÆ the structure's first system (FCC
  unchanged). `_ud_matrix_from_bnt` unchanged (cubic Cartesian holds for BCC).
- `orchestrator.py` `_ALL_111_PLANES` ŌåÆ `plane_normals(structure)`;
  `burgers_vectors(plane)` ŌåÆ `burgers_in_plane(structure, plane)`. The
  `*ŌłÜ2` reconstitutions (`orchestrator.py:927,1031,1047`; `viz/burgers.py:64`)
  ŌåÆ multiply by the actual `|b_int|` of the family (ŌłÜ2 for FCC Ō¤©110Ō¤®, ŌłÜ3 for
  BCC Ō¤©111Ō¤®) sourced from the registry, not a literal.
- `q_hkl` (4 sites): **unchanged in 4.3a** ŌĆö `q/|q|` is exact for cubic (incl.
  BCC). A regression test asserts BCC `q/|q|` equals `B┬ĘG/|B┬ĘG|` (cubic
  `B ŌłØ I`), documenting that 4.3b generalizes these.
- Provenance: emit `structure_type`, `slip_families`, `poisson_ratio`,
  `burgers_magnitude_um`, `material` next to the existing crystal attrs.

## 4. Stage 4.3b ŌĆö HCP (separate plan, after 4.3a lands)

Specified here at design level; refined into its own implementation plan once
4.3a is validated. The hard parts, all rooted in HCP being non-orthonormal:

1. **Non-orthonormal frame conversion.** Slip directions [uvw] ŌåÆ Cartesian via
   `A┬Ę[uvw]`; plane normals (hkl) ŌåÆ Cartesian via `B┬Ę(hkl)`; `t = b├Śn` in
   Cartesian. `_ud_matrix_from_bnt` and `burgers_in_plane`/`plane_normals`
   take the `UnitCell` and convert to Cartesian **before** normalizing/crossing
   (cubic path: A,B ŌłØ I, so 4.3a results are unchanged ŌĆö gate).
2. **Per-dislocation |b|.** HCP mixes Ō¤©aŌ¤® and Ō¤©c+aŌ¤® (very different
   magnitudes), so the scalar `b` in `Fd_find*`, `find_hg_*`, and the two
   numba kernels (`_population_hg_kernel`, `_scene_perdis_hg_kernel`) becomes a
   per-dislocation `b_arr`. This is a kernel-signature change; FCC/BCC pass a
   uniform array (bit-identical to the scalar path ŌĆö parity test).
3. **Non-cubic q_hkl.** Route the 4 `forward_model.py` sites through
   `mount.cell.B @ [h,k,l]` then normalize (reuse the `compute_omega_eta`
   pattern). Thread the cell/B into `Find_Hg` and the population builders.
   Cubic stays bit-identical.
4. **4-index MillerŌĆōBravais.** Accept HCP slip families in (hkil)/[uvtw] and
   convert to 3-index internally; registry entries for basal {0001}Ō¤©11-20Ō¤®,
   prismatic {10-10}Ō¤©11-20Ō¤®, pyramidal Ō¤©aŌ¤® and Ō¤©c+aŌ¤®.
5. **c/a dependence.** |b| and geometry depend on c/a (from the cell); covered
   by `burgers_magnitude` once it uses `A`.

DoD (4.3b): `[crystal] cif = "Ti.cif"` (or Mg) + an HCP reflection ŌåÆ forward
image + identify library with HCP Ō¤©aŌ¤®/Ō¤©c+aŌ¤® labels; FCC/BCC paths bit-identical.
